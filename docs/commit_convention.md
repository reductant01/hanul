# 커밋 메시지 규칙

커밋 제목은 아래 형식을 사용한다.

---

## 형식

```
<타입>(<범위>): <요약>
```

- **타입**: 이 커밋이 어떤 종류의 변경인지 (필수)
- **범위**: 선택. 변경이 일어난 모듈·폴더 (예: `controllers`, `config`, `scripts/hanul`)
- **요약**: 50자 내외, 동사 현재형, 마침표 생략. "무엇을 왜/어떻게 했는지" 한 문장

---

## 타입 정의 (언제 쓸지)

| 타입 | 의미 | 언제 쓰나 |
|------|------|-----------|
| **feat** | 기능 추가 | 사용자/시스템에 **새로 동작하는 것**을 넣을 때 (새 스크립트, 새 옵션, 새 모듈 등) |
| **fix** | 버그 수정 | **잘못 동작하던 것을 고칠 때** (오동작, 크래시, 잘못된 계산/로직 수정) |
| **docs** | 문서만 수정 | **코드가 아니라 문서만** 바꿀 때 (README, md, 주석만 수정, 문서 이동·정리) |
| **style** | 스타일·포맷 | **동작은 그대로**, 보기만 바꿀 때 (들여쓰기, 줄바꿈, 따옴표 통일, 불필요 공백 제거 등) |
| **refactor** | 리팩터링 | **동작 결과는 같게**, 구조·이름·위치만 바꿀 때 (폴더 분리, 함수 쪼개기, 변수/파일 이름 변경, 중복 제거) |
| **test** | 테스트 | 테스트 코드 추가·수정할 때 |
| **chore** | 잡일 | 빌드·설정·도구·기타 (의존성 추가, 설정 파일 수정, 폴더만 생성, .gitignore 등) |

---

## 구분 예시

- **feat** vs **fix**: 새로 "맵 저장 경로를 hanul 폴더로" 넣으면 `feat`. "맵이 안 저장되던 걸 고침"이면 `fix`.
- **docs**: `*.md` 수정, 문서 폴더 정리, 주석만 수정 → `docs`. (실제 코드 로직 수정은 feat/fix/refactor.)
- **style**: 동작은 전혀 안 바뀜. 포맷터 돌리거나 들여쓰기/공백만 맞춘 경우.
- **refactor**: "한울 전용 스크립트를 scripts/hanul로 옮김", "common 폴더로 공통 코드 분리", "함수 이름만 변경" → 동작은 그대로이면 `refactor`.
- **chore**: `config` 값만 수정, `.gitignore` 추가, 빈 폴더 추가, 패키지 버전 올리기 등.

---

## 예시

- `feat(nav2): loc 모드에 Nav2 bringup 연동`
- `fix(omni_odometry): Y축 delta_y 부호 보정`
- `docs(hanul): 02 맵핑 오류 표 추가`
- `style(common): 들여쓰기 4칸 통일`
- `refactor(controllers): webots/robot 폴더 분리, common 공통화`
- `chore(config): amcl_params update_min_a 0.02로 변경`
